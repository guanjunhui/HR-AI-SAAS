# 数据库测试数据完善计划

## 现状分析

| 表名 | 当前数据量 | 目标数据量/租户 | 需补充 |
|------|----------|--------------|-------|
| positions | 10 (仅 tenant_default) | 100+ / 每个租户 | 190+ |
| employees | 2 (tenant_default:2) | 100+ / 每个租户 | 200+ |
| employment_events | 0 | 100+ / 每个租户 | 200+ |
| headcounts | 0 | 100+ / 每个租户 | 200+ |

## 实施步骤

### 1. 生成职位数据 (positions)
- 为 tenant_default 补充 90+ 个职位
- 为 tenant_demo 创建 100+ 个职位
- 包含：技术类、产品类、设计类、运营类、市场类、财务类、人力资源类等

### 2. 生成员工数据 (employees)
- 为 tenant_default 生成 100+ 员工
- 为 tenant_demo 生成 100+ 员工
- 员工关联组织单位和职位
- 包含多种员工状态：trial（试用）、regular（正式）

### 3. 生成任职事件数据 (employment_events)
- 为 tenant_default 生成 100+ 任职事件
- 为 tenant_demo 生成 100+ 任职事件
- 包含多种事件类型：entry（入职）、regular（转正）、transfer（调岗）、promotion（晋升）、demotion（降级）、salary_change（调薪）、resignation（离职）
- 覆盖不同状态：draft（草稿）、pending（待审批）、approved（已通过）、rejected（已驳回）、cancelled（已取消）

### 4. 生成编制数据 (headcounts)
- 为 tenant_default 生成 100+ 编制记录
- 为 tenant_demo 生成 100+ 编制记录
- 关联组织单位和职位
- 设置预算人数和实际人数

## 技术方案

使用 Python 脚本生成 SQL INSERT 语句，通过 Docker exec 执行导入。

## 验收标准
- [ ] 每个租户职位数达到 100+
- [ ] 每个租员工数达到 100+
- [ ] 每个租户任职事件数达到 100+
- [ ] 每个租户编制数达到 100+
- [ ] 数据关联关系正确（员工→组织、员工→职位等）
